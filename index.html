02 — Calibration

Fit the model to your guitar

The textbook deflection model rarely matches a given guitar outright — string construction, fretting force and setup all shift how sharp each fretted note really goes, in either direction. Measure the actual intonation of the guitar as it stands now and the tool fits one deflection-gain factor that scales the whole model up or down to match. Every section below then uses it. (For this guitar both the measured break points and Mottola's calculator land well above gain 1 — it needs more compensation than textbook, not less.)

Deflection sharpening and saddle setback bend the error-vs-fret curve in almost exactly the same way (≈100 % collinear), so cents alone can't tell them apart. To break the tie, give the current break point of each string — the fore-aft distance from the nut to where the string actually leaves the old saddle (a ruler reading, not the crest height in §07). The cells are pre-filled with the measured break points of the current saddle; override them if you re-measure.

Then, with each string tuned open to pitch, fret it and read cents sharp on a tuner (strobe ideal) at a few frets — higher frets carry the most signal. Leave blanks where you didn't measure; any string with a break value and at least one fret counts. The fit also estimates one shared constant offset to soak up imperfect open tuning.

Gain 1 = textbook deflection. Below 1 = the guitar is flatter than predicted, so it needs less compensation; above 1 = sharper, needing more — both happen. (This guitar fits ~1.6, and Mottola's calculator independently lands ~1.9× the textbook setbacks.) Type a value directly to explore, or fit it from data. Because the break point is known, every fret then agrees on the same gain — the collinearity that defeats a cents-only fit now works in our favour.

Anchor the target saddle to your current saddle (recommended)

Your current saddle already intonates well, so instead of computing the target break points from the absolute model, take your measured break points and scale each by the action² ratio the model predicts between the as-built and target actions (target = current × REST/RES). The deflection gain cancels in that ratio, so the cut inherits all the real-world compensation baked into the working saddle and only adjusts it for the lower action. Off = pure model (the old behaviour). Needs a break value per string; the nut (if compensated) still follows the model.

07 — 3D model

OpenSCAD saddle

Turns the numbers above into a printable / CNC-able 3D saddle. The top profile is faired to a smooth radius-and-tilt curve through the per-string crest heights; each string's crown peaks exactly on its computed break point. Download the parametric .scad and open it in OpenSCAD to render or export an STL — every value sits in editable variables at the top of the file.

Blank

Thickness follows the saddle slot width and string spread follows §01 — change them there. Strings are centred on the blank, leaving equal margins at each end (~10.2 mm). End trims cut material off each end so the piece drops into a shorter or round-ended slot — the strings don't move, so keep each trim under the end margin. The two face trims thin the saddle from the nut side and the pin side independently (modeled thickness = slot width − both, now 3.0 mm). Because each crown bevels down from its break point, shaving the nut-side face cuts into the down-slope of the bass crowns (whose break sits at that face) — lowering those break edges and nudging the bass break a touch further back. Break-edge roundover rounds the sharp crown to a radius so the string beds on a smooth contact (0 = sharp peak; it lowers the crown slightly, by about 0.0 mm).

The model

Fretting a string deflects it down to the fret, raising tension and sharpening the note — the dominant intonation error, after Trevor Gore. A single saddle setback nulls only one fret, so compensation is split between the saddle (upper register) and the nut (low frets), and solved per string by least-squares fit of the error curve. Bending stiffness is excluded by default per Gore's findings; the toggle adds it back (the Elmendorp/Mottola view) for comparison.

String presets use D'Addario's published wound-string unit weights with core diameters from their core rule; plain-string masses are computed from steel density. Brands at the same gauge differ only marginally, so the data is shared where the gauge sets match. Replace any value with your own measurements for a precise fit. All computation runs locally; nothing is sent anywhere.
